## Naming and organization findings

- Directory casing is mixed by category: `Project` and `CV` use uppercase names, while `albums`, `albums-optimized`, and `data` use lowercase names.
- Current references match that casing, so this is an organization issue rather than a runtime defect.
- GitHub Pages is case-sensitive. Case-only renames on Windows are high risk and require a deliberate Git rename procedure.
- Spaces and non-ASCII characters are common in image filenames. Current encoded paths work, but path changes require URL encoding and production verification.
- JPG/WebP pairs in `assets/Project/` and the source/optimized Gallery directories are intentional pipeline pairs, not safe duplicate-removal candidates.
- Profile and SEO images are mixed with other asset categories: the portrait is at the root of `assets/`, while the shared SEO image is also a project source image.

## Risks

- Moving a project WebP without updating Excel can cause regenerated JSON to restore the old broken path.
- Updating Excel without regenerating JSON can leave the deployed site inconsistent with the workbook.
- Moving Gallery images before resolving the dual flow can break the Excel path, generated JSON, legacy JSON, and optimization pipeline differently.
- Case-only directory renames may appear successful on Windows but fail after deployment to Linux-based GitHub Pages.
- Renaming files containing spaces or non-ASCII characters can produce mismatched encoded and decoded URLs.
- Moving CV PDFs can break both hard-coded HTML downloads and Excel-generated link data.
- Moving the shared high-throughput JPG can break Open Graph and Twitter previews across all public pages.
- Removing source JPG files because WebP versions exist would break future optimization/regeneration.
- Moving `assets/data/` requires coordinated renderer and deployment changes across every data-driven page.

## Recommended next action

Do not move any asset immediately.

The Gallery runtime flow is unified and the unused general CV files are archived. The next recommended phase is CV data synchronization and regeneration. Profile/SEO image normalization should come later as a coordinated, low-volume path migration.
